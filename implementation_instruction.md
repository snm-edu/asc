# ヒューマン・アナトミー・アトラス2026 iPadアプリ 実装指示書

## 1. 目的

札幌看護医療専門学校に入学する高校生が、看護師・視能訓練士・歯科衛生士・臨床工学技士で必要となる解剖学を、**一体型の高精度3D解剖体**を操作しながら学べるiPadアプリを実装する。

> 重要方針: 学習領域は職種別に分類表示できるが、3Dモデル自体は分離しない。  
> **1つの人体解剖モデルに全領域が統合されていることを必須要件**とする。

## 2. 対象ユーザー

- 札幌看護医療専門学校へ進学予定の高校生
- 解剖学初学者
- iPadユーザー（授業・自宅学習）

## 3. プラットフォーム・基本仕様

- 対応OS: iPadOS（最新 + 2世代前まで）
- 対応向き: 横向き固定（推奨）
- オフライン対応: 主要学習機能はオフライン利用可
- パフォーマンス目標
  - 通常表示: 30fps以上
  - 部位選択反応: 200ms以内目標

## 4. 3Dモデル実装要件

### 4.1 モデル構造

- 3Dモデルは**全身一体型**とし、以下を同一モデル内に保持すること
  - 骨格
  - 筋
  - 神経
  - 血管
  - 臓器
  - 眼関連詳細
  - 口腔・歯科関連詳細
  - 心肺・腎など臨床工学関連ランドマーク

### 4.2 領域分類の扱い

- 看護師 / 視能訓練士 / 歯科衛生士 / 臨床工学技士 の分類は、表示フィルタ・学習コース・クイズ出題範囲で切り替え可能にする
- ただし分類切替時も、同じ3D解剖体を使い続けること

### 4.3 正確性

- 実在解剖学に準拠した位置・形状・相対関係を維持
- 学科教員または医療監修者レビューを通過したモデルのみ採用

## 5. ジェスチャー操作要件

- 1本指ドラッグ: モデル回転
- 2本指ピンチ: 拡大 / 縮小
- 2本指ドラッグ: 平行移動
- 1本指タップ: 部位選択
  - 選択部位をハイライト表示
  - 部位名ラベルをポップアップ表示

## 6. 学習UI要件

### 6.1 画面構成

- 中央: 3D表示領域
- 上部: レイヤー切替・検索・領域フィルタ
- 下部: 部位情報（日本語名、英語名、説明）
- 側部: クイズモード切替、学習履歴

### 6.2 ラベル表示

- タップした部位に対して以下を表示
  - 日本語名称
  - 英語名称（ON/OFF）
  - 簡易説明

### 6.3 レイヤー

- 皮膚 / 筋 / 骨 / 神経 / 血管 / 臓器 を個別に表示ON/OFF
- レイヤー切替時にカメラ位置を維持し、連続学習を妨げない

## 7. 学習機能要件

- 名称学習モード
  - 任意部位をタップして名称確認
- クイズモード
  - 「この名称はどこ？」（部位選択）
  - 「この部位名は？」（名称選択）
- 進捗管理
  - 学習時間、正答率、苦手部位

## 8. データ設計要件

- 各部位にユニークIDを付与
- 部位情報テーブルに以下を保持
  - 部位ID
  - 日本語名
  - 英語名
  - 所属分類（看護 / 視能 / 歯科 / 臨床工学）
  - 説明文
  - 関連レイヤー

## 9. 受け入れ基準

1. 1つの統合人体モデルで全領域学習が可能であること
2. 1本指回転、2本指拡大縮小・平行移動、1本指タップ選択が安定動作すること
3. タップ選択で名称表示が正しく行われること
4. 領域フィルタを切り替えてもモデルの連続性が保たれること
5. 主要部位の名称・位置に重大な解剖学的誤りがないこと
